01

Create Product

Data simulator Ledger

Create a new product and store it on the blockchain. Each time a new product is created, a transaction with all product information is sent to the blockchain via a smart contract.

Once a product is created, a unique QR-code will be generated. This QR-code will be used to redirect customers to our web application which extract all product information from the blockchain.

AssetID

Unique identifier in the blockchain. Each entry has an unique ID.

AssetType

The type of asset, i.e. description. Used assets: product, productAsset, machineAsset

ProductID

UUID of the product, will be used by the consumer for looking up data

OrderID

Corresponding order number for the product.

TasksPerformed

Tasks already performed on this product.

TasksToPerform

List of tasks which still have to be performed, i.e. drilling a hole.

Throws

GatewayException, CommitException

02

Request Task

Data simulator Ledger

Request the next task to be performed on the product. Based on the assetID, the product entry will be fetched. The next task will be extracted from the TasksToPerform field.

AssetID

The assetID previously assigned. Stored on the product.

Throws

EndorseException, CommitException, SubmitException, CommitStatusException

03

Perform Task

Data simulator

Once a product got a task, the product will tell the machines the task needs to be performed. If a machine is idle, the product will claim this machine and lock it. Next, the task will be simulated. This concept makes this simulation product-based.

After the task is done, four transactions will be made. The first three are machineAsset transactions. These include information about the machine (which task is performed when and where) as well as sensor data (circumstances). The fourth transaction is a productAsset transaction. This transaction includes information about which task is performed on what product by which machine.

Note: This step will be repeated until all tasks are done.

Product

The product on which a task needs to be performed.

Throws

-

04

Update Product

Data simulator Ledger

After simulating performing a task, the product will be updated in the blockchain. The task that is performed will be moved from the tasksToPerform list to the tasksPerformed list.

Note: This step will be repeated until all tasks are done.

AssetID

The assetID previously assigned. Stored on the product.

Task

Performed task.

Throws

EndorseException, CommitException, SubmitException, CommitStatusException

05

Get Product Information

Backend Ledger

Get product information from the API. This will return a JSON object with all product information.

/getProductAssets<ID>

Get all productAssets based on UUID of the product.

/getMachineAssets<ID>

Get all machineAssets based on UUID of the product.

/getDataByHour<hour, owner>

Get all data points (temperature or humidity) within a certain hour.

/getQRCode<ID>

Get a QR code based on the UUID of the product.

Throws

GatewayException, ParseException

06

Display Product Information

Backend Frontend

Once all data is extracted from the blockchain in the backend, the data will be displayed in our web application. Here, the customer can see alle relevant product information.
